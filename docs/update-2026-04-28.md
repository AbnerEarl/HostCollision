# 更新日志 - 2026-04-28

## 概述

本次更新主要围绕 **碰撞结果精准过滤** 和 **大数据量性能优化** 两个方向，新增了碰撞结果与直接访问IP响应的联合比对过滤、基于 SimHash 的同IP相似内容去重、TXT 文件中文乱码修复，以及 IP 预检测并发数提升。

## 一、新增功能

### 1. 碰撞结果与直接访问IP响应的联合比对过滤

**问题背景**：碰撞成功的结果中，部分 Host+IP 的碰撞响应与直接访问该 IP（不带 Host 头）的响应完全一致，说明服务器并未根据 Host 头返回差异化内容，这类结果属于误报，应当丢弃。

**原有不足**：此前的过滤逻辑仅通过 `strings.Contains` 做内容包含检查、Title 比对、FNV hash 指纹比对和编辑距离相似度比对，**缺少状态码维度的联合比对**。当碰撞请求和直接访问 IP 的状态码、内容长度、Server 头、Body 内容完全一致时，应直接判定为无效结果。

**解决方案**：在碰撞核心逻辑 `collision()` 方法中，在现有的内容比对之前，新增**状态码 + 内容长度 + Server 头 + Body 内容**的四元组联合比对。

**实现细节**：
- 比对维度：`StatusCode`、`ContentLen`、`ServerHeader`、`AppBody()`
- 采用**分层过滤策略**：先比对轻量级的状态码/内容长度/Server头（O(1)），三者全部相同时才进行昂贵的 Body 字符串比较（O(n)）
- 过滤位置在所有现有比对逻辑之前，作为最早的快速排除层

**新增代码逻辑**：
```go
// 状态码+内容长度联合比对：碰撞结果与直接访问IP结果完全一致则丢弃
if newRequest.StatusCode == baseRequest.StatusCode &&
    newRequest.ContentLen == baseRequest.ContentLen &&
    newRequest.ServerHeader == baseRequest.ServerHeader {
    if newRequest.AppBody() == baseRequest.AppBody() {
        return collisionResultFailed
    }
}
```

**日志输出示例**：
```
协议:http://, ip:1.2.3.4, host:example.com 匹配失败-2(响应与直接访问IP完全一致)
```

**涉及文件**：
- `pkg/collision/collision.go` — `collision()` 方法新增联合比对逻辑

### 2. 基于 SimHash 的同IP相似内容去重

**问题背景**：不同 Host 与同一个 IP 碰撞时，可能获得高度相似的响应内容（如同一个默认页面、同一个 CDN 节点返回的通用页面），导致结果中出现大量重复信息。此前 SimHash 指纹仅在碰撞成功时计算并存储在结果中，**未实际用于去重**。

**解决方案**：在碰撞成功保存结果时，新增基于 `Protocol+IP` 维度的 SimHash 相似内容聚合去重。同一个 IP 下，如果多个 Host 的 SimHash 海明距离 ≤ 3（高度相似），只保留第一个结果。

**实现细节**：

**新增字段**：
| 字段 | 类型 | 位置 | 说明 |
|------|------|------|------|
| `simhashDedup` | `map[string]uint64` | `Worker` 结构体 | 全局共享的 SimHash 去重集合，key 为 `Protocol\|IP`，value 为首个成功结果的 SimHash |

**去重算法**：
1. 碰撞成功后，以 `Protocol|IP` 为 key 查找 `simhashDedup`
2. 如果该 key 已存在，计算当前结果的 SimHash 与已有 SimHash 的海明距离
3. 海明距离 ≤ 3 → 判定为高度相似内容，丢弃当前结果
4. 海明距离 > 3 → 内容差异较大，保留当前结果（不更新已有 SimHash）
5. 如果该 key 不存在，记录当前 SimHash 作为基准

**线程安全**：`simhashDedup` 的读写操作在 `resultsMu` 互斥锁保护下执行，与 `resultDedup` 共享同一把锁，确保并发安全。

**修改 `NewWorker` 函数签名**：
```go
// 修改前
func NewWorker(..., resultDedup map[string]struct{}, ...) *Worker

// 修改后
func NewWorker(..., resultDedup map[string]struct{}, simhashDedup map[string]uint64, ...) *Worker
```

**所有入口函数均已更新**：

| 入口函数 | 文件 | 创建 `simhashDedup` | 传入 `NewWorker` |
|----------|------|:--:|:--:|
| `RunWithOptions` | `hostcollision.go` | ✅ | ✅ |
| `RunWithCallback` | `hostcollision.go` | ✅ | ✅ |
| `main()` | `cmd/hostcollision/main.go` | ✅ | ✅ |

**日志输出示例**：
```
协议:http://, ip:1.2.3.4, host:example.com 碰撞成功但SimHash与已有结果高度相似(距离≤3),跳过
```

**涉及文件**：
- `pkg/collision/collision.go` — `Worker` 结构体新增 `simhashDedup` 字段；`NewWorker` 新增参数；`collision()` 方法添加 SimHash 去重检查
- `hostcollision.go` — `RunWithOptions` 和 `RunWithCallback` 创建共享 `simhashDedup` map 并传入 Worker
- `cmd/hostcollision/main.go` — `main()` 创建共享 `simhashDedup` map 并传入 Worker

## 二、Bug 修复

### 1. TXT 文件中文乱码修复（新增 UTF-8 BOM）

**问题描述**：CLI 模式下输出的 TXT 文件在部分编辑器（如 Windows 记事本、Excel 等）中打开时，中文内容（如 Title）显示为乱码。

**根因分析**：CSV 文件已在创建时写入了 UTF-8 BOM（`0xEF, 0xBB, 0xBF`），但 TXT 文件遗漏了此处理。部分编辑器在没有 BOM 标记的情况下，默认使用系统编码（如 GBK）解析文件，导致 UTF-8 编码的中文内容显示为乱码。

**修复方案**：在 TXT 文件创建后，写入 UTF-8 BOM 标记。

**修改位置**：`cmd/hostcollision/main.go` 中 TXT 文件创建逻辑。

```go
// 修复前
txtFile, err = os.Create(outputPath + ".txt")

// 修复后
txtFile, err = os.Create(outputPath + ".txt")
// 写入 BOM（支持编辑器中文显示）
txtFile.Write([]byte{0xEF, 0xBB, 0xBF})
```

| 场景 | 修复前 | 修复后 |
|------|--------|--------|
| Windows 记事本打开 TXT | ❌ 中文可能乱码 | ✅ 正常显示 |
| Excel 打开 TXT | ❌ 中文可能乱码 | ✅ 正常显示 |
| macOS/Linux 终端查看 | ✅ 正常 | ✅ 正常 |

**涉及文件**：
- `cmd/hostcollision/main.go` — TXT 文件创建后写入 UTF-8 BOM

## 三、性能优化

### 1. IP 预检测并发数提升（20 → 100）

**问题背景**：在大数据量场景下（如 8000 个 IP），IP 预检测阶段（检测 IP 端口可达性和协议支持情况）是碰撞前的必要步骤。原有并发数为 20，对于 8000 个 IP 的预检测耗时过长。

**优化方案**：将 IP 预检测的并发信号量从 20 提升到 100。

**修改位置**：`pkg/collision/collision.go` 中 `PreDetectIPs` 函数。

```go
// 修改前
sem := make(chan struct{}, 20)

// 修改后
sem := make(chan struct{}, 100)
```

**性能对比**（以 8000 个 IP、平均 5 秒/IP 超时为例）：

| 指标 | 修改前（20并发） | 修改后（100并发） | 提升 |
|------|:--:|:--:|:--:|
| 预检测耗时 | ~2000s（33分钟） | ~400s（6.7分钟） | **5倍** |
| 内存开销 | 极低 | 低（仅增加 goroutine 栈） | 可忽略 |
| 网络压力 | 低 | 中等 | 可接受 |

**涉及文件**：
- `pkg/collision/collision.go` — `PreDetectIPs` 函数并发信号量从 20 调整为 100
